# doc-to-onepage

**将"复杂信息"转化为"清晰决策表达"的AI能力层**

***

## 一句话定义

将任意文档转化为\*\*"可决策的一页纸（OnePage）"\*\*，并生成可直接用于汇报的网页与飞书文档。

**帮你把"信息"变成"说服力"。**

***

## 核心价值

本技能不是简单的格式转换工具，而是一个**汇报内容重构引擎**，提供三大核心能力：

### 1. 内容重构（Content Reframing）

- 从"信息堆积" → "结论优先"
- 自动提炼：标题、核心结论、关键要点、数据支撑

### 2. 信息压缩（High-Density Compression）

- 将长文压缩为：**1页**可读完、**3分钟**可讲清
- 保留关键决策信息，删除冗余内容

### 3. 汇报表达增强（Presentation Enhancement）

- 汇报开场句（Opening）
- 结论总结句（Summary）
- Q\&A预判（关键问题提示）

***

# 📐 OnePage 通用设计规范（Design System）

***

## 1️⃣ 页面结构规范（必须遵守）

```
[Header - 结论区]
[Core - 核心逻辑区（最多3块）]
[Execution - 执行区（可选）]
[Risk - 风险区（建议必有）]
[Appendix - 附录区（折叠）]
```

***

## 2️⃣ 信息密度规范（核心约束）

**📏 硬性规则：**

- 总阅读时长：≤ 3分钟
- 核心模块数：≤ 3
- 每模块要点：≤ 3条
- 每条字数：≤ 40字

***

## 3️⃣ 结论区规范（最高优先级）

**必须包含：** 标题 + 一句话结论 + CTA

***

## 4️⃣ CTA规范（强制）

**每个 OnePage 必须有：** 是否批准 / 是否推进 / 是否分配资源

***

# 🎬 OnePage 动效规范

**🧠 一句话原则：** 动效只做一件事——控制阅读顺序，强化重点

***

## ✅ 基础规则（必须遵守）

- **时长：** 200–400ms
- **一屏：** 只用1种动效
- **只在：** 内容进入时、重点强调时 使用

***

## 🎯 必备4种动效

| 动效             | 用途             |
| -------------- | -------------- |
| **Fade In**    | 所有内容默认出现方式     |
| **Stagger**    | 卡片依次出现（间隔80ms） |
| **Hover Lift** | 卡片轻微上浮（+阴影）    |
| **Collapse**   | 控制信息密度（附录用）    |

***

## 🧭 标准阅读动线（配合动效）

```
结论 → 问题 → 方案 → 收益 → 风险
```

- **首屏：** 结论优先出现
- **滚动：** 模块逐块 fade in

***

## 🎨 场景差异

| 场景       | 动效风格            |
| -------- | --------------- |
| **决策汇报** | 极简，几乎无感动效       |
| **项目推进** | 有节奏（列表/时间线逐步出现） |
| **宣传海报** | 可强化（但≤3种动效）     |

***

## ⚠️ 红线（踩了就降级）

- ❌ 动效 > 内容
- ❌ 动画过慢
- ❌ 多种动画混用

***

## 🚀 加分项（选做）

- 关键结论轻微高亮（一次即可）
- Before / After 对比动效（强感知价值）

***

## 🎯 最终一句话

**好的动效让用户"顺着看完并做决定"，而不是"停下来欣赏动画"。**

***

# 🎨 场景模板规范（8种模板）

***

## 🟦 1. 决策汇报型 (decision-report)

**🎯 目标：** 让老板拍板

**🧱 结构：**

```
结论 → 方案对比 → 推荐方案 → 收益 → 风险
```

**⚙️ 规则：** 必须有方案A/B对比 + 推荐理由（3条以内）
**🎨 风格：** 商务蓝、极简、强对比

***

## 🟩 2. 产品方案型 (product-solution)

**🎯 目标：** 对齐认知

**🧱 结构：**

```
背景（Why）→ 方案（What）→ 用户价值（Value）→ 实现方式（How）
```

**⚙️ 规则：** 必须有用户视角
**🎨 风格：** 中性色、信息密集

***

## 🟨 3. 项目推进型 (project-progress)

**🎯 目标：** 同步进度 + 控风险

**🧱 结构：**

```
当前状态 → 关键进展 → 问题/风险 → 下一步
```

**⚙️ 规则：** 必须有时间节点 + Owner
**🎨 风格：** 清单式、时间线

***

## 🟥 4. 治理专项型 (governance-improvement)

**🎯 目标：** 解决长期问题

**🧱 结构：**

```
问题拆解 → 根因分析 → 治理方案 → 长期机制
```

**⚙️ 规则：** 必须有根因 + 长期策略

***

## 🟧 5. 大促保障型 (holiday-support)

**🎯 目标：** 保稳定

**🧱 结构：**

```
目标 → 保障策略 → 关键指标 → 应急预案
```

**⚙️ 规则：** 必须有风险预案 + 监控指标

***

## 🟪 6. 产品/工具型 (product-tool)

**🎯 目标：** 解决效率问题

**🧱 结构：**

```
问题（Pain）→ 工具方案（Tool）→ 使用方式 → 收益
```

***

## 🟫 7. 项目管理型 (project-management)

**🎯 目标：** 长期能力建设

**🧱 结构：**

```
愿景（Vision）→ 路径（Roadmap）→ 阶段目标 → 资源
```

***

## ⚫ 8. 团队说明型 (team-intro)

**🎯 目标：** 快速认知团队

**🧱 结构：**

```
我们是谁（Who）→ 做什么（What）→ 怎么找（How）→ 合作方式
```

***

## 模板推荐机制

**AI自动推荐 + 用户确认或修改**

***

## 工作流

```
第一阶段：理解内容
第二阶段：重构内容（提炼 + 匹配模板）
第三阶段：生成与优化
```

***

## 交互流程（Agent 必须遵循）

### 1️⃣ 启动 → 2️⃣ 模板推荐 → 3️⃣ 信息补充 → 4️⃣ 文档处理 → 5️⃣ 内容分析 → 6️⃣ 生成初稿 → 7️⃣ 创建飞书文档 → 8️⃣ 交互式优化（循环） → 9️⃣ 生成网页版本 → 🔟 最终交付
